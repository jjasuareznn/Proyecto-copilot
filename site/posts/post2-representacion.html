Representación de Grafos

Publicado: 2025-12-02 • Autor: Curso de Estructuras de Datos

Dos representaciones principales

Los grafos suelen representarse en memoria de dos maneras principales: lista de adyacencia y matriz de adyacencia.

1) Lista de Adyacencia

Es una colección (por ejemplo, un array o un diccionario) donde cada vértice guarda la lista de vecinos. Es eficiente en espacio para grafos dispersos (pocos ejes comparados con el número máximo posible).

Ejemplo (para el grafo de 5 nodos A,B,C,D,E):

Adj = {
  A: [B, D],
  B: [A, C, D],
  C: [B, E],
  D: [A, B, E],
  E: [D, C]
}

Si usamos estructuras de lenguaje: un diccionario en Python o un objeto con arrays en JavaScript.

2) Matriz de Adyacencia

Se usa una matriz n x n donde la celda (i, j) indica si existe la arista entre i y j. Consume O(n^2) espacio y es conveniente para consultar rápidamente si una arista existe.

Ejemplo:

A B C D E
A01010
B10110
C01001
D11001
E00110

Comparativa

  • Lista de adyacencia: Espacialmente eficiente para grafos dispersos; permite iterar vecinos rápidamente.
  • Matriz de adyacencia: Ofrece consulta O(1) para ver si existe una arista; es mejor para grafos densos o para algoritmos que necesitan consultas rápidas.
